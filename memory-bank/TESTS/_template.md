# Test: [Что тестируется]

Задокументированный тест-кейс. Используется для проверки утверждений из `FACTS/` или поведения компонентов.

## Contract

- when: нужно описать тест; зафиксировать результат теста; проверить утверждение из FACTS/
- prereq: есть конкретное поведение или факт, который нужно верифицировать
- reads: этот файл; связанный `FACTS/<related_fact>.md` если указан
- writes: при обновлении результата или статуса
- success: тест однозначно описан; результат зафиксирован; связь с фактом установлена
- on_fail: если тест не прошёл -> обновить `test_status: failed`; вынести в `OPEN_QUESTIONS.md`

## Canonical scope

- contains: цель теста, входные данные, шаги, ожидаемый/фактический результат
- excludes:
  - стратегия тестирования -> `TESTING.md`
  - конкретный код тестов -> кодовая база
