Обнови memory bank проекта. Выполни шаги точно.

## 0. Privacy scan

**Перед любой записью:** просканируй контент сессии на теги приватности:
- `<private>...</private>` → **удалить** весь контент внутри тега, заменить на `[redacted]`
- `<sensitive pii_risk="high">...</sensitive>` → записать без тега, пометить `pii_risk: high` в frontmatter файла
- `<ephemeral>...</ephemeral>` → записать **только** в `.local/`; НЕ промотировать

Алгоритм: `PATTERNS/privacy-control.md`.

## 1. Обнови CURRENT.md

Открой `memory-bank/.local/CURRENT.md`:
- Установи "Последнее обновление" на текущий момент с идентификатором агента.
- Отметь завершённые пункты. Добавь новые.
- Удали пункты, выполненные более одной сессии назад.
- Обнови список активных файлов.
- Установи checkpoint compaction на текущий момент.
- Жёсткий лимит: 80 строк. Если длиннее — суммаризируй старое.

## 2. Обнови HANDOFF.md

Перепиши `memory-bank/.local/HANDOFF.md` с нуля по структурированному контракту:

```markdown
# Handoff — [ГГГГ-ММ-ДД ЧЧ:ММ]

## Request
[Что просил пользователь; 1-2 предложения]

## Investigated
[Что исследовано; ключевые файлы и модули — одна строка]

## Learned
[Ключевые находки; факты, которые стоит помнить]

## Completed
[Что конкретно завершено; результаты]

## Next steps
[1-3 приоритетных действия для следующей сессии]

## Risks
[Подводные камни, блокеры, что может пойти не так]

## Active files
[Изменённые или активно используемые файлы — одна строка]
```

Жёсткий лимит: 40 строк. Если не помещается — сократи Investigated и Active files до одной строки.

## 3. Промоция устойчивых знаний

Проверь сессию на знания для промоции:

**Решения** → Добавь строку в `memory-bank/DECISIONS.md`. Если нетривиальное — создай ADR.
Заполни колонки `Conf` и `Источник` согласно `PATTERNS/provenance-standard.md`.

**Паттерны** → Создай или обнови файл в `memory-bank/PATTERNS/`.
К значимым фактам добавь inline-аннотацию: `<!-- prov: ГГГГ-ММ-ДД | conf: observed | src: SESSIONS/файл.md -->`.

**Изменения архитектуры** → Обнови соответствующую секцию `ARCHITECTURE.md`.
Обнови дату "Верифицировано". К значимым фактам добавь `<!-- prov: ... -->`.

**Новые конвенции** → Обнови `CONVENTIONS.md`.
К новым правилам добавь `<!-- prov: ГГГГ-ММ-ДД | conf: observed | src: SESSIONS/файл.md -->`.

**При создании FACT** → Определи `observation_type` и `concepts` из `PATTERNS/observation-typing.md`.
Добавь в frontmatter нового FACTS/*.md файла.

## 4. Проверка соответствия конституции

Если в сессии были приняты архитектурные решения или изменения:
- Прочитай `memory-bank/CONSTITUTION.md`.
- Проверь, что каждое решение соответствует принципам.
- Если обнаружено несоответствие — НЕ промотируй решение автоматически.
  Вместо этого добавь в отчёт: «⚠️ Решение [X] потенциально конфликтует с принципом [Y]. Требуется ревью человека.»

## 5. Сессионная заметка (если задача значительная)

Создай `memory-bank/.local/SESSIONS/ГГГГ-ММ-ДД-<агент>-<slug>.md` по шаблону.

## 6. Отчёт

Перечисли что обновлено. Отметь отложенные промоции.

## Правила

- Никогда не записывай секреты, токены или PII. Применяй privacy scan (шаг 0) перед каждой записью.
- Один факт в одном файле — ссылайся, не дублируй.
- Каждый новый или обновлённый факт получает "Верифицировано: [дата]".
- Если файл превышает лимит строк — суммаризируй или вынеси в ARCHIVE/.
