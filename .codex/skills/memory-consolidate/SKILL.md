Консолидируй недавние сессионные заметки в стабильные файлы памяти.

> **Триггер рефлексии:** хук `Stop` автоматически проверит, нужна ли рефлексия, при завершении сессии. Не запускай `/memory-reflect` вручную из consolidate — это вызовет двойную обработку.

## Процесс

### 1. Собери кандидатов
Прочитай файлы в `memory-bank/.local/SESSIONS/` с неотмеченными пунктами в секции "Знания для промоции".

### 2. Privacy scan
**Перед обработкой каждого кандидата** просканируй на теги приватности:
- `<private>` → удалить контент перед записью
- `<ephemeral>` → пропустить при промоции (не промотировать в стабильные файлы)
- `<sensitive>` → записать с `pii_risk: high`

Алгоритм: `PATTERNS/privacy-control.md`.

### 3. Для каждого кандидата — реши

Перед решением ADD выполни три шага конфликт-детекта:

**Шаг A — Извлеки subject:** определи тему факта в 1–3 слова (что описывает этот факт).
Пример: "база данных", "механизм авторизации", "деплой-платформа".

**Шаг B — Найди в целевом файле:** поищи упоминания того же subject.
Если не найдено → решение **ADD**, перейди к шагу 4.

**Шаг C — Сравни (если найдено):**

| Результат сравнения | Решение |
|---------------------|---------|
| Факты идентичны или эквивалентны | **SKIP** |
| Новый уточняет / расширяет старый | **UPDATE** → C.1 |
| Факты прямо противоречат | **CONFLICT** → C.2 |
| Нарушает принцип из `CONSTITUTION.md` | **CONSTITUTION_CONFLICT** → C.3 |

**C.1 — UPDATE Decision Protocol:**
1. Определи, какой факт новее (по дате сессии).
2. Определи, какой confidence выше.
3. При UPDATE сохраняй **ВЫСШИЙ** confidence из двух.
4. Объедини provenance: добавь новый источник к существующему.

**C.2 — CONFLICT Auto-Resolution (3 измерения):**
1. **Confidence:** `confirmed` побеждает `observed`; `inferred` побеждает `observed`.
2. **Recency:** при равном confidence — более новый факт предпочтительнее.
3. **Breadth:** при равном confidence и recency — факт с большим числом источников надёжнее.
4. Если все 3 измерения дают **одного победителя** → автоматический UPDATE с audit trail:
   ```
   <!-- conflict-resolved: ГГГГ-ММ-ДД | winner: [новый/старый] | reason: [confidence|recency|breadth] -->
   ```
5. Если результат **неоднозначен** → НЕ перезаписывай. Запиши в `OPEN_QUESTIONS.md § Conflicts`.

**При CONFLICT (неоднозначный)** — запиши в `memory-bank/OPEN_QUESTIONS.md § Conflicts`:

```markdown
### ⚠️ CONFLICT — ГГГГ-ММ-ДД

**Subject:** [тема]
**Существующий факт:** `[файл:строка]` — "[текст]" (conf: [level], verified: [date])
**Новый факт:** `[SESSIONS/файл.md]` — "[текст]" (conf: [level])
**Auto-resolution attempt:**
- Confidence: [существующий/новый/равный]
- Recency: [существующий/новый/равный]
- Breadth: [существующий/новый/равный]
**Result:** unresolved → human review
**Рекомендация:** [обновить / уточнить / выбрать один]
```

**C.3 — CONSTITUTION_CONFLICT** — НЕ применять. Записать полный контекст в `OPEN_QUESTIONS.md § Conflicts` с пометкой 🚫 CONSTITUTION_CONFLICT. Только для human review.

### 4. Извлеки observation_type и concepts
Для каждого наблюдения из сессии определи тип и концепты согласно `PATTERNS/observation-typing.md`.
Используй `observation_type` как подсказку для выбора канонического файла-владельца (маршрут из таблицы типов).

### 5. Примени изменения
Для каждого ADD или UPDATE:
- Запиши в канонический файл (по правилам владения из INDEX.md).
- Добавь "Верифицировано: [сегодня]".
- К значимым фактам добавь провенанс: `<!-- prov: ГГГГ-ММ-ДД | conf: observed | src: SESSIONS/файл.md -->`.
- В `DECISIONS.md` заполни колонки `Conf` и `Источник`.
- При создании FACTS/*.md — добавь `observation_type` и `concepts` в frontmatter.
- Отметь пункт промоции в сессионном файле.

### 6. Очисти сессии
- Файлы со всеми отмеченными пунктами и старше 14 дней → перенеси в `ARCHIVE/`.
- Файлы старше 30 дней без исключений → перенеси в `ARCHIVE/`.
- Файлы в `ARCHIVE/` старше 90 дней → предложи удаление.

### 7. Отчёт

```
## Консолидация — [дата]

### Промотировано
- [факт] → [файл] (ADD/UPDATE)

### Конфликты — авторазрешены
- [subject] → winner: [новый/старый] | reason: [confidence|recency|breadth]

### Конфликты — требуют ревью
- [противоречащие факты и их источники]

### Архивировано
- [N] сессионных файлов перемещено

### Осталось
- [N] файлов с необработанными пунктами
```
